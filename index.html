GoConvey

All Systems Go

Testing is awesome again

GoConvey makes test-driven development in Go a genuine pleasure. Enjoy the luxuries of auto-test, colored terminal output, reporting in your browser, behavior-driven tests, a huge suite of assertions, a clean DSL, and more.

Use by running go test

Since GoConvey is directly integrated with Go's testing package, you can keep using your existing native Go tests or use GoConvey's natural, easy-to-use composition style. Many people use GoConvey just for the web UI.

Get started

Assuming your GOPATH is set, cd to your project's folder and run:

$ go get github.com/smartystreets/goconvey $ $GOPATH/bin/goconvey

Then open your browser to localhost:8080.

Brilliant Web UI

Test results update automatically

The GoConvey server watches all folders under a given "watched" directory, and when a file changes in those folders, tests are automatically run and the page is updated.

You can manually invoke a re-run of the tests by clicking the icon in the top-right corner.

At-a-glance overview, or details if you want

View the immediate summary just by glancing at the bar across the top of the page. Green is passing, yellow is failed test, red is panic, and gray is build failed.

Dig in to the story view of the page for details on tests, whether they passed or failed. You can collapse or ignore entire packages if they get in your way.

Gracefully examine failing tests

Failures bubble to the top

Failing tests conveniently appear at the top of the page so you can quickly find and fix any problems. Many built-in assertions, such as ShouldEqual, provide a handy-dandy diff tool so you can see what's going on.

Shortcut links to get around

Click links to scroll to the test in context, or to open the file in Sublime Text at the exact line of the failing assertion. You can expand and collapse folders on the left side for links to shortcut to encapsulating Test... function names.

Resolve panics and build failures

This is what a panic feels like

Imagine the little Golang gopher running around in its wheel and suddenly, it starts to panic. Fortunately, GoConvey makes it easy to recover your little gopher.

Build failure output

Sometimes more frustrating than failed tests and even panics are build failures, because they're usually unexpected. GoConvey brings the output of go test to your browser so you can quickly pinpoint the compilation problems.

HTML5 desktop notifications

Heck, why waste screen space?

Even though GoConvey's web UI is responsive, sometimes you still want your editor to fill the screen. Or maybe you can't be bothered to take your eye off your code, even for a second. (GoConvey is written by fellow gophers. We understand!)

Enable desktop notifications by clicking the icon in the web UI and you can get test results in whatever window you have open.

Notifications that aren't annoying

Notifications disappear after about 3 seconds: just enough time for you to get the gist of the test results. If you need more details, your browser tab is waiting for you.

Your notification preference is remembered using HTML5's localStorage feature, so you don't have to keep toggling them on and off.

Responsive design for limited screen space

Plenty of room for everything

GoConvey can squish alongside your editor if you have limited screen space and still want to see your test output. Plus, when test results update, your scroll position will be maintained when the screen is redrawn.

Test code generator

Write tests with natural language

Write GoConvey tests by typing your program's expected behavior, BDD-style. Don't worry about Go syntax in here, because that will be automatically stubbed out for you.

Smart input preserves tabs

Tab indentation is used to nest Convey() statements, and the textarea supports tab preservation on newlines, so you can focus purely on your program's behavior and tests.
